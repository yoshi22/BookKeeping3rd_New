# ワイヤーフレーム設計

## 1. 設計原則

### 1.1 レイアウト原則
- **シンプル第一**: 必要な情報のみを表示し、認知負荷を最小化
- **一貫性**: 同じ情報は常に同じ位置・形式で表示
- **階層性**: 重要度に応じた情報の配置・サイズ調整
- **アクセシビリティ**: WCAG 2.1 AA準拠のタッチターゲット・コントラスト

### 1.2 画面サイズ対応
- **プライマリ**: iPhone SE（375×667px）以上
- **セカンダリ**: 大画面Android（400×800px）程度
- **タッチターゲット**: 最小44×44px、推奨48×48px以上

### 4.1 画面サイズ別レイアウト調整

**iPhone SE (375px幅)**:
- 余白: 16px
- フォントサイズ: 16px (本文)、20px (見出し)
- ボタン高さ: 48px
- カード間隔: 12px

**大画面 (400px幅以上)**:
- 余白: 20px
- フォントサイズ: 18px (本文)、24px (見出し)
- ボタン高さ: 52px
- カード間隔: 16px

## 5. アクセシビリティ対応

### 5.1 フォーカス管理
```
Tab Order (フォーカス順序):
1. 戻るボタン
2. 問題文 (音声読み上げ対象)
3. 選択肢A → B → C → D
4. 解答ボタン
5. タブバー (左から右へ)
```

### 5.2 色覚対応
```
Color Scheme (色設計):
正解: Green (#4CAF50) + ✅マーク
不正解: Red (#F44336) + ❌マーク
警告: Orange (#FF9800) + ⚠️マーク
情報: Blue (#2196F3) + ℹ️マーク

High Contrast Mode:
背景: #000000 (黒)
文字: #FFFFFF (白)
強調: #FFFF00 (黄)
```

### 5.3 音声読み上げ対応
```
Accessibility Labels:
<問題文> aria-label="問題: 田中商店は商品100,000円を現金で仕入れた。この取引の仕訳を示しなさい。"

<選択肢> aria-label="選択肢A: 借方 仕入 100,000円、貸方 現金 100,000円"

<ボタン> aria-label="解答を確定します"

<進捗> aria-label="第1問 15問中5問目を解答中"
```

## 6. ワイヤーフレーム検証項目

### 6.1 ユーザビリティ検証
- [ ] 重要な情報が3秒以内に認識可能
- [ ] 主要アクションが1タップで実行可能
- [ ] エラー状態からの回復手順が明確
- [ ] 学習の継続性が阻害されない設計

### 6.2 アクセシビリティ検証
- [ ] 44×44px以上のタッチターゲット確保
- [ ] 4.5:1以上のコントラスト比
- [ ] 音声読み上げでの理解可能性
- [ ] キーボードのみでの操作可能性

### 6.3 技術制約検証
- [ ] React Native コンポーネントでの実装可能性
- [ ] オフライン動作時の表示内容
- [ ] 画面サイズ・向き変更への対応
- [ ] パフォーマンス要件との整合性
